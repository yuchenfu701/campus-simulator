## 🚀 部署步骤（Railway方案）

### 第一步：准备MongoDB Atlas（云数据库）

1. **注册MongoDB Atlas账号**
   - 访问：https://www.mongodb.com/cloud/atlas/register
   - 使用邮箱注册（免费）

2. **创建免费集群**
   - 登录后点击 "Create" → "Build a Database"
   - **重要：选择 "Free" 套餐（不要选M10或其他付费套餐！）**
   - Free套餐特点：
     - 512 MB存储空间
     - RAM共享
     - VCPU共享
     - 完全免费，适合学习和小型应用
   - 选择云服务商和地区（选择离你最近的）
   - 点击 "Create Deployment"（创建部署）

3. **配置数据库访问**
   - 创建数据库用户：
     - Username: `campus_admin`
     - Password: 设置一个强密码（记住它！）
   - 配置网络访问：
     - 点击 "Network Access" → "Add IP Address"
     - 选择 "Allow Access from Anywhere"（0.0.0.0/0）
     - 点击 "Confirm"

4. **获取连接字符串**
   - 点击 "Connect" → "Connect your application"
   - 复制连接字符串，格式如下：
     ```
     mongodb+srv://campus_admin:<password>@cluster0.xxxxx.mongodb.net/campus_simulator?retryWrites=true&w=majority
     ```
   - 将 `<password>` 替换为你设置的密码
   - **保存这个连接字符串，后面会用到**

---

### 第二步：部署到Railway

1. **注册Railway账号**
   - 访问：https://railway.app/
   - 使用GitHub账号登录（最简单）

2. **创建新项目**
   - 登录后点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 如果没有GitHub仓库，选择 "Empty Project"

3. **上传代码**
   
   **方法A：使用GitHub（推荐）**
   - 在GitHub创建新仓库
   - 将 `server` 文件夹的内容上传到仓库
   - 在Railway中选择这个仓库

   **方法B：直接上传（简单）**
   - 在Railway中选择 "Empty Project"
   - 点击 "Add Service" → "GitHub Repo"
   - 或者使用Railway CLI上传

4. **配置环境变量**
   - 在Railway项目页面，点击 "Variables" 标签
   - 添加以下环境变量：
     ```
     MONGODB_URI = mongodb+srv://campus_admin:你的密码@cluster0.xxxxx.mongodb.net/campus_simulator?retryWrites=true&w=majority
     CORS_ORIGIN = *
     NODE_ENV = production
     ```
   - Railway会自动设置 `PORT` 变量

5. **部署**
   - Railway会自动检测到 `package.json` 并开始部署
   - 等待部署完成（约2-3分钟）
   - 部署成功后，Railway会提供一个域名，如：`https://your-app.up.railway.app`

6. **获取API地址**
   - 在Railway项目页面，点击 "Settings"
   - 找到 "Domains" 部分
   - 复制你的域名（如：`https://your-app.up.railway.app`）
   - **API地址就是：`https://your-app.up.railway.app/api`**

---

### 第三步：配置前端使用云端服务器

1. **打开主菜单页面** (`main-menu.html`)

2. **添加服务器配置功能**
   - 在页面加载时检查是否有配置的服务器地址
   - 如果没有，提示用户配置

3. **配置服务器地址**
   - 打开浏览器控制台（F12）
   - 输入以下代码：
     ```javascript
     apiClient.setServerURL('https://your-app.up.railway.app');
     ```
   - 将 `your-app.up.railway.app` 替换为你的Railway域名
   - 按回车执行

4. **验证连接**
   - 刷新页面
   - 打开浏览器控制台（F12）
   - 应该看到：`🌐 使用配置的服务器地址: https://your-app.up.railway.app/api`
   - 尝试使用论坛、好友等功能，应该可以正常工作

## 📝 注意事项

### 1. MongoDB Atlas免费限制
- 免费套餐有512MB存储限制
- 适合小型应用使用
- 如果数据量大，考虑升级

### 2. Railway免费额度
- 每月$5免费额度
- 超出后需要付费
- 小型应用通常不会超出

### 3. 安全性
- 生产环境建议设置 `CORS_ORIGIN` 为你的前端域名
- 不要在前端代码中硬编码API密钥
- 定期更新依赖包

### 4. 域名配置
- Railway提供免费域名（`.up.railway.app`）
- 可以绑定自定义域名（需要付费）

## 🆘 常见问题

### Q: 部署后无法连接数据库？
A: 检查MongoDB Atlas的网络访问设置，确保允许所有IP访问（0.0.0.0/0）

### Q: CORS错误？
A: 检查 `CORS_ORIGIN` 环境变量，确保包含你的前端域名

### Q: 如何更新代码？
A: 如果使用GitHub，推送代码后Railway会自动重新部署

### Q: 如何查看日志？
A: 在Railway项目页面点击 "Deployments" → 选择部署 → 查看日志
