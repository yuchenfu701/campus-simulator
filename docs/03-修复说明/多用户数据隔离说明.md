# 👥 多用户数据隔离实现说明

## 🎯 需求背景

### 用户反馈
> "每个账号要保存信息，新的账号要重置为初始信息"

### 原问题
- **所有账号共享数据**：使用固定的 `student_001` ID
- **新账号无初始化**：登录后看到的是其他账号的数据
- **数据混乱**：A账号的积分、装备会被B账号看到

---

## ✅ 解决方案

### 核心思路
1. **每个账号独立ID**：基于手机号生成唯一用户ID
2. **数据隔离存储**：每个用户在 `localStorage` 中有独立的数据空间
3. **新用户自动初始化**：首次登录创建默认数据
4. **全局统一管理**：所有页面使用当前登录用户的ID

---

## 🔧 技术实现

### 1. 用户ID生成系统 (`login.html`)

#### 登录时生成用户ID
```javascript
// 基于手机号生成唯一ID
const userId = 'user_' + phone;  // 例如: user_13812345678

// 保存用户信息（包含userId）
const userData = {
    userId: userId,           // ← 新增：唯一用户ID

    phone: phone,
    avatar: userAvatar || '👤',

    userType: 'student'

localStorage.setItem('campus_user', JSON.stringify(userData));
```

### 2. 新用户数据初始化 (`login.html`)

#### initUserPointsData 函数
```javascript
function initUserPointsData(userId, nickname) {
    // 1. 获取或创建积分系统数据结构
    let pointsData = localStorage.getItem('campus_points_system');
    
    if (!pointsData) {
        // 首次使用系统，创建空数据结构
        pointsData = {
            students: {},  // 所有学生数据的容器
            config: {}
        };
    } else {
        pointsData = JSON.parse(pointsData);
    }
    
    // 2. 检查该用户是否已存在
    if (!pointsData.students[userId]) {
        // 新用户 - 创建初始数据
        pointsData.students[userId] = {
            id: userId,
            name: nickname,
            points: 100,           // 初始积分：100
            totalEarned: 100,
            inventory: [],         // 空背包
            avatar: {              // 默认形象
                hair: 'none',
                glasses: 'none',
                hat: 'none'
            },
            achievements: [],      // 空成就
            tasks: {
                daily: {},
                weekly: {}
            }
        };
        
        // 保存到localStorage
        localStorage.setItem('campus_points_system', JSON.stringify(pointsData));
        console.log(`✅ 新用户 ${nickname} (${userId}) 数据已初始化`);
    } else {
        // 老用户 - 加载现有数据
        console.log(`👤 用户 ${nickname} (${userId}) 数据已加载`);
    }
}
```

## 📊 数据流程图

### 用户注册/登录流程
```
用户输入手机号和昵称
        ↓
生成用户ID (user_手机号)
        ↓
保存到 localStorage['campus_user']
        ↓
检查是否为新用户
        ↓
    是新用户？
    ├── 是 → 初始化积分数据（100积分，空背包）
    └── 否 → 加载现有数据
        ↓
跳转到主菜单
```

### 功能页面数据加载流程
```
打开任意功能页面
        ↓
读取 localStorage['campus_user']
        ↓
获取 userId
        ↓
创建 PointsSystem 实例
        ↓
调用 setCurrentUser(userId)
        ↓
加载该用户的专属数据
        ↓
显示该用户的积分/装备/任务等
```

## 🐛 常见问题

### Q1: 如果手机号重复怎么办？
**A:** 相同手机号会生成相同的 userId，会加载到同一份数据。这是预期行为，相当于"账号登录"。

### Q2: 快速登录会产生新账号吗？
**A:** 是的。每次快速登录都会生成随机手机号，创建新账号。如果想复用账号，请使用正常登录。

### Q3: 数据存储在哪里？
**A:** 存储在浏览器的 localStorage 中，清除浏览器数据会丢失。

### Q4: 如何删除某个用户的数据？
**A:** 
```javascript
// 在浏览器控制台
const pointsData = JSON.parse(localStorage.getItem('campus_points_system'));
delete pointsData.students['user_13812345678'];
localStorage.setItem('campus_points_system', JSON.stringify(pointsData));
```

### Q5: 如何导出/备份用户数据？
**A:**
```javascript
// 导出所有数据
const backup = {
    user: localStorage.getItem('campus_user'),
    points: localStorage.getItem('campus_points_system')
};
console.log(JSON.stringify(backup, null, 2));
// 复制输出，保存到文件
```
