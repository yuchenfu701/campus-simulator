# 🌐 公网部署指南 - 让所有人通过网址访问你的游戏

本指南将帮助你将游戏部署到互联网，让任何人可以直接通过网址访问，无需下载文件或启动本地服务器。

---

## 🎯 推荐方案对比

| 方案 | 优点 | 缺点 | 适合场景 |
|------|------|------|----------|
| **GitHub Pages** ⭐ | 完全免费、简单、稳定 | 只能部署静态文件 | 纯前端项目 |
| **Vercel** ⭐⭐ | 免费、快速、自动HTTPS | 需要GitHub账号 | 前端+后端API |
| **Netlify** ⭐⭐ | 免费、功能强大、易用 | 需要GitHub账号 | 前端+后端API |
| **Railway** | 支持后端服务器 | 有免费额度限制 | 需要后端功能 |

---

## 🚀 方案一：GitHub Pages（最简单，推荐）

### 优点
- ✅ **完全免费**，无限制
- ✅ **简单易用**，几分钟完成
- ✅ **稳定可靠**，GitHub官方服务
- ✅ **自动HTTPS**，安全访问
- ✅ **全球CDN**，访问速度快

### 部署步骤

#### 第一步：准备GitHub账号

1. **注册GitHub账号**（如果还没有）
   - 访问：https://github.com/
   - 点击 "Sign up" 注册
   - 验证邮箱

#### 第二步：创建GitHub仓库

1. **创建新仓库**
   - 登录GitHub后，点击右上角 "+" → "New repository"
   - Repository name: `campus-simulator`（或你喜欢的名字）
   - Description: "爱哲安民未来学校校园模拟器"
   - 选择 **Public**（公开，GitHub Pages需要）
   - **不要**勾选 "Initialize this repository with a README"
   - 点击 "Create repository"

#### 第三步：上传代码到GitHub

**方法A：使用GitHub Desktop（最简单，推荐）**

1. **下载GitHub Desktop**
   - 访问：https://desktop.github.com/
   - 下载并安装

2. **登录并添加仓库**
   - 打开GitHub Desktop
   - 登录你的GitHub账号
   - 点击 "File" → "Add Local Repository"
   - 点击 "Choose..." 选择你的项目文件夹（`d:\编程\test`）
   - 点击 "Add repository"

3. **提交并上传**
   - 在GitHub Desktop中，你会看到所有文件
   - 在左下角输入提交信息："Initial commit - 校园模拟器"
   - 点击 "Commit to main"
   - 点击 "Publish repository"
   - 等待上传完成（可能需要几分钟）

**方法B：使用网页上传（无需安装）**

1. **压缩项目文件**
   - 将项目文件夹压缩成zip文件
   - 注意：**不要包含** `node_modules`、`.git` 等文件夹

2. **上传到GitHub**
   - 在GitHub仓库页面，点击 "uploading an existing file"
   - 将压缩包解压后的文件拖拽上传
   - 输入提交信息："Initial commit"
   - 点击 "Commit changes"
   - 等待上传完成

**方法C：使用命令行（如果你熟悉Git）**

```bash
cd d:\编程\test
git init
git add .
git commit -m "Initial commit - 校园模拟器"
git branch -M main
git remote add origin https://github.com/你的用户名/campus-simulator.git
git push -u origin main
```

#### 第四步：启用GitHub Pages

1. **进入仓库设置**
   - 在GitHub仓库页面，点击 "Settings"（设置）标签
   - 在左侧菜单中找到 "Pages"

2. **配置Pages**
   - Source: 选择 "Deploy from a branch"
   - Branch: 选择 "main"（或 "master"）
   - Folder: 选择 "/ (root)"
   - 点击 "Save"

3. **配置自定义404页面（重要）**
   - GitHub Pages会自动使用 `404.html` 作为404页面
   - 我们已经创建了 `404.html`，它会自动重定向到 `site.html`
   - 这样访问根路径时会显示网站首页

4. **获取网站地址**
   - 等待几分钟，GitHub会显示你的网站地址
   - 格式：`https://你的用户名.github.io/campus-simulator`
   - 例如：`https://zhangsan.github.io/campus-simulator`

5. **访问网站**
   - **直接访问根路径**：`https://你的用户名.github.io/campus-simulator`
   - 会自动显示 `site.html` 作为首页
   - 或者直接访问：`https://你的用户名.github.io/campus-simulator/site.html`
   - 如果显示404，等待几分钟后刷新

#### 第五步：配置自定义域名（可选）

如果你有自己的域名：

1. **在GitHub Pages设置中添加域名**
   - 在Pages设置页面，找到 "Custom domain"
   - 输入你的域名（如：`campus.example.com`）
   - 点击 "Save"

2. **配置DNS**
   - 在你的域名服务商（如阿里云、腾讯云）添加DNS记录
   - 类型：CNAME
   - 主机记录：`campus`（或 `@`）
   - 记录值：`你的用户名.github.io`

## 🚀 方案二：Vercel（推荐，功能强大）

### 优点
- ✅ **免费且快速**
- ✅ **自动HTTPS**
- ✅ **全球CDN**
- ✅ **自动部署**（每次GitHub推送自动更新）
- ✅ **支持后端API**

### 部署步骤

#### 第一步：准备GitHub仓库

按照"方案一"的步骤创建GitHub仓库并上传代码。

#### 第二步：部署到Vercel

1. **注册Vercel账号**
   - 访问：https://vercel.com/
   - 点击 "Sign Up"
   - 选择 "Continue with GitHub"（使用GitHub登录）

2. **导入项目**
   - 登录后，点击 "Add New..." → "Project"
   - 在 "Import Git Repository" 中找到你的仓库
   - 点击 "Import"

3. **配置项目**
   - Framework Preset: 选择 "Other" 或 "Static"
   - Root Directory: 留空（使用根目录）
   - Build Command: 留空（不需要构建）
   - Output Directory: 留空
   - 点击 "Deploy"

4. **获取网站地址**
   - 等待部署完成（约1-2分钟）
   - Vercel会自动提供一个域名，如：`campus-simulator.vercel.app`
   - 你也可以在项目设置中配置自定义域名

5. **自动部署**
   - 以后每次你推送代码到GitHub，Vercel会自动重新部署
   - 无需手动操作

## 🚀 方案三：Netlify（功能强大）

### 优点
- ✅ **免费且功能强大**
- ✅ **拖拽部署**（最简单）
- ✅ **自动HTTPS**
- ✅ **表单处理**（如果需要）

### 部署步骤

#### 方法A：拖拽部署（最简单）

1. **访问Netlify**
   - 访问：https://www.netlify.com/
   - 注册账号（使用GitHub登录）

2. **拖拽部署**
   - 登录后，在首页找到 "Want to deploy a new site without connecting to Git?"
   - 将你的项目文件夹拖拽到页面上的区域
   - 等待上传和部署完成

3. **获取网站地址**
   - Netlify会自动提供一个域名，如：`random-name-12345.netlify.app`
   - 你可以在 "Site settings" → "Change site name" 修改域名

#### 方法B：GitHub连接部署（推荐）

1. **连接GitHub**
   - 登录Netlify后，点击 "Add new site" → "Import an existing project"
   - 选择 "Deploy with GitHub"
   - 授权Netlify访问你的GitHub账号
   - 选择你的仓库

2. **配置部署**
   - Build command: 留空
   - Publish directory: `.`（当前目录）
   - 点击 "Deploy site"

3. **自动部署**
   - 以后每次推送代码，Netlify会自动重新部署

## 🔧 部署后配置

### 1. 配置后端服务器（如果需要）

如果你的游戏需要后端功能（论坛、好友等），还需要部署后端服务器：

**参考文档：** `docs/02-功能说明/云端部署指南.md`

**快速步骤：**
1. 部署后端到Railway（参考云端部署指南）
2. 获取后端API地址，如：`https://your-app.up.railway.app`
3. 在游戏中使用 `server-config.html` 配置API地址

### 2. 设置默认首页

确保 `site.html` 作为网站首页：

**GitHub Pages：**
- 在仓库根目录创建 `index.html`，内容重定向到 `site.html`
- 或直接在GitHub Pages设置中指定首页

**Vercel/Netlify：**
- 已配置 `vercel.json` 和 `netlify.toml`，会自动处理

## 🐛 常见问题

### Q1: 部署后显示404错误？

**解决方案：**
1. 等待几分钟（GitHub Pages需要时间）
2. 检查文件名是否正确（区分大小写）
3. 确保 `site.html` 文件已上传
4. 检查GitHub Pages设置是否正确

### Q2: 图片或资源无法加载？

**解决方案：**
1. 检查文件路径是否正确（使用相对路径）
2. 确保所有资源文件都已上传
3. 检查浏览器控制台错误信息

### Q3: 网站更新后没有变化？

**解决方案：**
1. 清除浏览器缓存（Ctrl+F5）
2. 等待几分钟（CDN缓存更新）
3. 检查代码是否已正确提交

### Q4: 移动端显示不正常？

**解决方案：**
1. 检查HTML中的viewport设置
2. 测试响应式设计
3. 使用移动端浏览器测试
