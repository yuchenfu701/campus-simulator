# 🌐 网站使用指南

## 📋 简介

现在你的校园模拟器游戏已经可以作为一个完整的网站运行了！通过简单的HTTP服务器，你可以在浏览器中访问所有功能，就像访问普通网站一样。

---

## 🚀 快速启动

### 方法一：使用启动脚本（推荐）

**Windows系统：**
1. 双击运行 `启动网站服务器.bat`
2. 等待服务器启动（会自动检测Python或Node.js）
3. 浏览器会自动打开网站首页
4. 如果没有自动打开，手动访问：`http://localhost:8080/site.html`

**Mac/Linux系统：**
```bash
python3 启动网站服务器.py
```

### 方法二：手动启动Python服务器

```bash
# Python 3
python -m http.server 8080

# 或 Python 2
python -m SimpleHTTPServer 8080
```

### 方法三：使用Node.js服务器

```bash
# 安装http-server（首次使用）
npm install -g http-server

# 启动服务器
http-server -p 8080
```

## 🔧 配置说明

### 端口配置

默认端口是 `8080`，如果需要修改：

**Python服务器：**
```python
# 编辑 启动网站服务器.py
PORT = 8080  # 改为你想要的端口
```

**Node.js服务器：**
```bash
http-server -p 3000  # 使用3000端口
```

### 服务器地址

- **本地访问**: `http://localhost:8080`
- **局域网访问**: `http://你的IP地址:8080`
  - Windows: 在命令行运行 `ipconfig` 查看IP
  - Mac/Linux: 在终端运行 `ifconfig` 查看IP

## 🌍 部署到互联网

如果你想让其他人也能访问你的网站，可以：

### 方法一：使用内网穿透工具

1. **ngrok**（推荐）
   ```bash
   # 安装ngrok
   # 下载: https://ngrok.com/
   
   # 启动你的本地服务器
   python 启动网站服务器.py
   
   # 在另一个终端运行ngrok
   ngrok http 8080
   ```
   
   然后你会得到一个公网地址，比如：`https://xxxxx.ngrok.io`

2. **花生壳**（国内推荐）
   - 注册账号：https://hsk.oray.com/
   - 下载客户端
   - 配置内网穿透

### 方法二：部署到云服务器

1. **GitHub Pages**（免费）
   - 将项目上传到GitHub
   - 启用GitHub Pages
   - 访问地址：`https://你的用户名.github.io/项目名`

2. **Vercel**（免费）
   - 注册：https://vercel.com/
   - 连接GitHub仓库
   - 自动部署

3. **Netlify**（免费）
   - 注册：https://www.netlify.com/

## ⚠️ 注意事项

### 1. 数据存储

- 游戏数据存储在浏览器的 **LocalStorage** 中
- 清除浏览器缓存会丢失数据
- 建议定期导出数据备份

### 2. 浏览器兼容性

推荐使用现代浏览器：
- ✅ Chrome（推荐）
- ✅ Edge
- ✅ Firefox
- ✅ Safari
- ❌ IE（不支持）

### 3. 后端服务器

如果需要使用论坛、好友等在线功能：
1. 启动后端服务器（参考 `快速启动指南.md`）
2. 确保MongoDB已运行
3. 访问 `http://localhost:3000/api/health` 检查服务器状态

### 4. 安全提示

- 本地服务器仅供开发和测试使用
- 部署到公网前，请确保做好安全防护
- 不要在生产环境使用默认端口

## 🐛 常见问题

### 问题1：端口被占用

**错误信息：** `Address already in use`

**解决方案：**
1. 关闭占用8080端口的程序
2. 或修改脚本中的端口号

### 问题2：无法访问网站

**检查清单：**
1. ✅ 服务器是否已启动？
2. ✅ 浏览器地址是否正确？
3. ✅ 防火墙是否阻止了连接？
4. ✅ 端口号是否正确？

### 问题3：页面显示404

**原因：** 文件路径不正确

**解决方案：**
1. 确保所有HTML文件都在项目根目录
2. 检查文件名是否正确（区分大小写）
3. 使用相对路径访问页面

### 问题4：功能无法使用

**可能原因：**
1. 后端服务器未启动（论坛、好友等功能需要）
2. JavaScript文件路径错误
3. 浏览器控制台有错误信息

**解决方案：**
1. 打开浏览器开发者工具（F12）
2. 查看Console标签页的错误信息
3. 根据错误信息修复问题
