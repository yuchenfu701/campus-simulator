# 历史小游戏集成说明

## 问题排查

如果点击"KARDS历史卡牌"游戏后无法进入，请按以下步骤排查：

### 1. 检查文件夹是否存在

确认以下文件夹存在：
- `新建文件夹 (5)` （原文件夹）
- 或者 `history-game` （如果运行了重命名脚本）

### 2. 检查文件路径

打开浏览器开发者工具（F12），查看控制台（Console）中的错误信息：
- 如果显示"404 Not Found"，说明路径不正确
- 如果显示"CORS错误"，说明需要本地服务器运行

### 3. 解决方案

#### 方案A：使用原文件夹名（当前默认）

代码默认使用 `新建文件夹 (5)/index.html` 路径，如果文件夹存在，应该可以直接打开。

#### 方案B：重命名文件夹（推荐）

1. 双击运行 `重命名历史游戏文件夹.bat`
2. 文件夹会被重命名为 `history-game`
3. 打开 `campus-games.html`，找到第170行左右
4. 将 `possiblePaths` 数组的顺序调换：
   ```javascript
   const possiblePaths = [
       'history-game/index.html',     // 改为第一个
       '新建文件夹 (5)/index.html'    // 改为第二个
   ];
   ```

### 4. 测试方法

1. 直接在浏览器中打开：`新建文件夹 (5)/index.html`
   - 如果能正常打开，说明文件没问题
   - 如果打不开，检查文件是否存在

2. 检查浏览器控制台：
   - 按 F12 打开开发者工具
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页，看哪个文件加载失败

### 5. 常见问题

**Q: 点击后没有任何反应？**
A: 打开浏览器控制台（F12），查看是否有JavaScript错误

**Q: 显示404错误？**
A: 检查文件夹名称是否正确，路径中是否有空格或特殊字符

**Q: 在iframe中无法加载？**
A: 尝试直接在新标签页打开，检查是否是iframe限制问题

## 文件结构

历史小游戏的文件结构：
```
新建文件夹 (5)/
├── index.html      # 主菜单
├── game.html       # 游戏页面
├── style.css       # 样式文件
├── script.js       # 游戏逻辑
└── menu.js         # 菜单逻辑
```
