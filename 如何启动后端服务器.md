# 🚀 如何启动后端服务器

## ⚠️ 错误提示说明

如果你看到 **"服务器不可用，请检查后端服务是否启动"** 的错误，说明后端服务器还没有启动。

---

## 📋 启动步骤（3种方法）

### 方法1：使用启动脚本（最简单）⭐推荐

1. **双击运行** `启动服务器.bat` 文件
2. 等待服务器启动完成
3. 看到 "🚀 服务器启动成功" 的提示即可

**优点：** 一键启动，自动检查环境

---

### 方法2：使用命令行（手动启动）

#### 步骤1：打开命令行

- **Windows**: 按 `Win + R`，输入 `cmd`，回车
- 或者在项目文件夹中，按住 `Shift` 右键，选择"在此处打开PowerShell窗口"

#### 步骤2：进入server目录

```bash
cd server
```

#### 步骤3：检查依赖（首次运行）

```bash
npm install
```

#### 步骤4：启动服务器

```bash
npm start
```

#### 步骤5：确认启动成功

看到以下信息说明启动成功：
```
🚀 服务器启动成功
📡 监听端口: 3000
🌐 API地址: http://localhost:3000/api
✅ MongoDB连接成功
```

---

### 方法3：使用开发模式（代码修改后自动重启）

```bash
cd server
npm run dev
```

**优点：** 修改代码后自动重启服务器

---

## 🔧 前置要求检查

### 1. 检查Node.js是否安装

打开命令行，输入：
```bash
node -v
```

如果显示版本号（如 `v18.17.0`），说明已安装 ✅

如果没有安装：
- 下载地址：https://nodejs.org/
- 选择 LTS 版本下载安装

### 2. 检查MongoDB Compass是否运行

1. 打开 **MongoDB Compass**
2. 连接到本地 MongoDB（默认：`mongodb://localhost:27017`）
3. 如果连接失败，检查MongoDB服务是否启动

**如何检查MongoDB服务：**
- Windows: 打开"服务"（services.msc），查找"MongoDB"服务
- 如果服务未运行，启动它

## 🐛 常见问题解决

### 问题1：端口3000被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方法：**

**方法A：关闭占用端口的程序**
```bash
# Windows PowerShell
netstat -ano | findstr :3000
# 找到PID后，结束进程
taskkill /PID <PID号> /F
```

**方法B：修改端口**
1. 在 `server` 文件夹创建 `.env` 文件
2. 添加内容：
   ```
   PORT=3001
   ```
3. 重启服务器

### 问题2：MongoDB连接失败

**错误信息：**
```
❌ MongoDB连接失败
```

**解决方法：**

1. **检查MongoDB Compass是否运行**
   - 打开MongoDB Compass
   - 确保能连接到 `mongodb://localhost:27017`

2. **检查连接字符串**
   - 在 `server` 文件夹创建 `.env` 文件（如果不存在）
   - 添加内容：
     ```
     MONGODB_URI=mongodb://localhost:27017/campus_simulator
     ```

3. **检查MongoDB服务**
   - Windows: 打开"服务"，查找"MongoDB"服务
   - 如果服务未运行，右键启动

### 问题3：依赖安装失败

**错误信息：**
```
npm ERR! ...
```

**解决方法：**

1. **清除缓存**
   ```bash
   npm cache clean --force
   ```

2. **使用国内镜像（如果网络慢）**
   ```bash
   npm config set registry https://registry.npmmirror.com
   ```

3. **重新安装**
   ```bash
   cd server
   rm -rf node_modules
   npm install
   ```

### 问题4：找不到npm命令

**解决方法：**
- 重新安装Node.js
- 确保安装时勾选"添加到PATH"

## ✅ 启动成功检查

### 1. 检查服务器是否运行

打开浏览器，访问：
```
http://localhost:3000/api/health
```

应该看到：
```json
{
  "success": true,
  "message": "服务器运行正常",
  "timestamp": "..."
}
```

### 2. 检查MongoDB连接

服务器启动时应该看到：
```
✅ MongoDB连接成功
📊 数据库: campus_simulator
```

### 3. 测试论坛功能

1. 打开 `campus-forum.html`
2. 点击"发布新帖子"
3. 输入内容并发布
4. 如果成功，说明服务器正常运行 ✅

## 📝 启动后的操作

### 保持服务器运行

- **不要关闭** 命令行窗口（服务器在运行）
- 如果需要停止服务器，按 `Ctrl + C`

### 查看日志

服务器运行时会显示：
- API请求日志
- 数据库操作日志
- 错误信息

### 后台运行（可选）

如果需要后台运行服务器：

**Windows（使用PM2）：**
```bash
npm install -g pm2
cd server
pm2 start server.js
pm2 save
pm2 startup
```

## 💡 提示

1. **首次启动**：需要先运行 `npm install` 安装依赖
2. **保持运行**：服务器需要一直运行，关闭窗口会停止服务
3. **修改代码**：使用 `npm run dev` 可以自动重启
4. **查看文档**：更多信息查看 `server/README.md`

## 🆘 还是不行？

如果按照以上步骤还是无法启动，请检查：

1. **Node.js版本**：建议使用 Node.js 16+ 版本
2. **防火墙**：确保3000端口没有被防火墙阻止
3. **权限问题**：以管理员身份运行命令行
4. **查看错误信息**：仔细阅读命令行中的错误提示
